## 🔄 工作流

```
选题 → 研究 → 审稿 → 美化 → 发布
  ↑                               |
  └── 反馈 ───────────────────────┘
```

1. 用户提出研究需求（GitHub Issues）
2. 主编制定计划，分派任务给探针
3. 探针完成多源研究、分析、交叉验证
4. 读者独立审稿，提交反馈
5. 调色板美化，生成 HTML 报告
6. 发布到 GitHub Pages + Release

## 🏷️ 报告质量标准

- 每个重要事实 ≥ 3 个独立来源交叉验证
- 来源分级：论文(A) > 官方博客(B) > 技术媒体(C) > 社区讨论(D)
- 区分事实、推断、观点、猜测
- 包含 Executive Summary + 技术深度分析 + 实践建议
- 每篇报告 3000-5000 字，含 .md + .html 双格式
