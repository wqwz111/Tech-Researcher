📋 Executive Summary

ACP（Agent Client Protocol）是 OpenClaw 连接外部编程智能体的桥梁。通过 ACP，你可以从聊天窗口直接调度 Codex、Claude Code、Gemini CLI 等专业编程 Agent，实现代码开发、测试修复、代码审查的自动化。ACP 支持一次性运行（run）和持久化会话（session）两种模式，配合线程绑定机制，在 Discord/Telegram 中实现持续的编程对话。resumeSessionId 功能让你可以在不同设备间无缝继续编程会话。

配置线程绑定

{
  acp: {
    enabled: true,
    backend: "acpx",
    defaultAgent: "codex",
    allowedAgents: ["pi", "claude", "codex", "opencode", "gemini", "kimi"],
  },
  session: {
    threadBindings: { enabled: true, idleHours: 24, maxAgeHours: 0 }
  },
  channels: {
    discord: {
      threadBindings: { enabled: true, spawnAcpSessions: true }
    },
    telegram: {
      threadBindings: { spawnAcpSessions: true }
    }
  }
}

🔄 会话恢复（resumeSessionId）

ACP 支持跨设备恢复会话。比如你在电脑上开始一个 Codex 编程会话，然后可以在手机上继续：

{
  "task": "Continue where we left off — fix remaining test failures",
  "runtime": "acp",
  "agentId": "codex",
  "resumeSessionId": "previous-session-id"
}
💡 适用场景：
  • 从笔记本切换到手机继续编程
  • Gateway 重启后恢复中断的会话
  • 将交互式 CLI 会话转为 headless 模式

🔐 权限管理

ACP 会话非交互式运行（没有 TTY），所以权限提示无法弹出确认。通过 permissionMode 配置：

模式行为适用场景
approve-all自动批准所有操作完全信任的 Agent
approve-reads只批准读操作，写/执行需提示默认，平衡安全
deny-all拒绝所有权限提示只做分析不修改
openclaw config set plugins.entries.acpx.config.permissionMode approve-all
openclaw config set plugins.entries.acpx.config.nonInteractivePermissions deny
openclaw gateway restart
