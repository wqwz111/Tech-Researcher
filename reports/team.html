🎯 项目理念
🎯

求真求实

每个重要事实 ≥ 3 个独立来源交叉验证。来源分级：论文(A) > 官方博客(B) > 技术媒体(C) > 社区讨论(D)。明确区分事实、推断、观点、猜测。

🔗

引用铁律

信息必须最新 — 优先 2024-2025 年来源
引用必须带链接 — 所有参考来源附 URL
禁止文本示意图 — 使用 Mermaid 或信息图

🤝

开放协作

通过 GitHub Issues 收集选题建议和报告反馈。欢迎社区参与审稿和讨论。

📊

持续改进

每次审稿都是改进机会。主编关注 Issues，收集有用信息，推动团队持续提升报告质量。
