Executive Summary

AI 编排平台（Orchestration Platforms）是连接 LLM、工具、数据源和业务逻辑的中间层，让开发者可以通过可视化拖拽或声明式配置来构建 AI 工作流，而无需从零编写底层集成代码。

本报告对比分析 5 个主流平台：LangFlow（LangChain 可视化包装）、Flowise（低代码 LLM 应用构建器）、Dify（开源 LLMOps 平台）、Coze（字节跳动 Bot 平台）、n8n（通用自动化，AI 增强）。

核心结论:

  • 技术团队 / 需要深度定制: LangFlow（基于 LangChain）或 Dify（自托管完整）
  • 快速搭建 Bot / 无代码需求: Coze（多渠道发布）或 Flowise（简单拖拽）
  • 已有自动化工作流 / 业务集成: n8n（连接 400+ 服务 + AI 节点）

    • 1. 平台概览

    LangFlow

  • 起源: 基于 LangChain 的可视化 UI（2023）
  • 技术栈: Python (FastAPI) + React
  • 定位: LangChain 工作流的可视化编辑器
  • 许可: MIT
  • 特色: 直接对应 LangChain 概念，可导出为 Python 代码
  • 适合: 熟悉 LangChain 的开发者，快速原型设计

    • Flowise

  • 起源: 开源低代码 LLM 应用构建器（2023）
  • 技术栈: Node.js + React
  • 定位: 拖拽式 LLM 工作流构建
  • 许可: Apache 2.0 (v1) / 商业 (v2)
  • 特色: 简单直观、节点丰富、支持 Agent
  • 适合: 非技术用户、快速原型

    • Dify

  • 起源: 开源 LLMOps 平台（2023，中国团队）
  • 技术栈: Python (Flask) + Next.js
  • 定位: 一站式 LLMOps（Prompt IDE + 工作流 + 知识库 + API）
  • 许可: 开源（部分商业功能）
  • 特色: 功能全面、支持知识库管理、多模型管理
  • 适合: 需要完整 LLMOps 能力的团队

    • Coze（扣子）

  • 起源: 字节跳动 Bot 开发平台（2023）
  • 技术栈: 托管 SaaS
  • 定位: 低代码 Bot 构建与多渠道发布
  • 许可: 云服务（免费使用）
  • 特色: 一键发布到抖音/飞书/微信等、插件生态、工作流编排
  • 适合: 面向消费者的 Bot、国内用户

    • n8n

  • 起源: 开源自动化平台（2019，后增加 AI 能力）
  • 技术栈: Node.js + Vue
  • 定位: 通用工作流自动化 + AI 节点
  • 许可: Fair-code (社区版免费)
  • 特色: 400+ 服务集成、成熟的自动化能力、AI Agent 节点
  • 适合: 业务自动化、多系统集成

    6. 中国市场特殊考量

    对于面向中国市场的项目，需要额外考虑：

    1. 国内模型支持: Dify 和 Coze 原生支持通义千问、文心一言、智谱 GLM 等国内模型

    2. 数据合规: 自建部署（Dify）可以完全控制数据出境

    3. 渠道发布: Coze 直接支持抖音、飞书、微信等国内渠道

    4. 网络访问: LangFlow/Flowise/n8n 依赖的海外 LLM 可能需要代理

    5. 社区支持: Dify 和 Coze 有活跃的中文社区
