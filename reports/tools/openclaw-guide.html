OpenClaw 深度使用指南

最后更新: 2026-03-14 | 分类: AI Agent 平台

Executive Summary

OpenClaw 是一个开源的 AI Agent 编排平台，提供 Agent 运行时、多渠道接入、技能系统和 MCP 集成。定位偏向 Agent 的生产运行时——关注会话管理、多渠道消息路由、持久化记忆和技能动态加载。

1. 架构概览

核心抽象

概念说明
AgentAI 代理实例，拥有独立配置、记忆和技能
Channel消息渠道（Telegram、Discord、Signal 等）
Session会话上下文，管理交互状态
Skill可复用的能力模块
MCPModel Context Protocol 工具集成

2. Agent 编排与会话管理

Agent 配置

agent:
  name: my-agent
  model: openrouter/anthropic/claude-sonnet-4
  system_prompt: "你是一个专业的技术顾问..."
  memory:
    backend: file       # 或 redis
    max_tokens: 4000
  tools: [web_search, read, write, exec]
  skills: [/path/to/skills/]

会话隔离

  • Direct: 用户私聊，独立会话
  • Group: 群组共享（可配置隔离）
  • Thread: 线程级隔离（Telegram topics）

3. 多渠道接入

4. 技能系统与 MCP 集成

技能目录结构

skills/
├── web-research/
│   ├── SKILL.md          # 技能描述
│   ├── references/       # 参考文档
│   └── scripts/          # 辅助脚本
├── code-review/
│   └── SKILL.md
└── weather/
    └── SKILL.md

MCP 配置

mcp:
  servers:
    - name: filesystem
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem"]
    - name: postgres
      command: npx
      args: ["-y", "@modelcontextprotocol/server-postgres"]

5. 高级用法与最佳实践

记忆管理

memory:
  backend: redis          # 生产推荐
  max_history: 50
  summarization: true
  summary_interval: 20

安全实践

  1. API Key 用环境变量，不硬编码
  2. 设置 allowed_chats 限制访问
  3. 限制 exec 工具的命令白名单
  4. 输入长度和内容检查

调试命令

openclaw gateway status
openclaw logs --follow
openclaw sessions list
openclaw sessions export <session-id>
